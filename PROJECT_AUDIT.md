# Mango Kids Project Audit

## Project concept

This is a preschool website built with React 19 and TanStack Start. It has public marketing pages, a Supabase-powered enquiry form and gallery, authentication, and an admin dashboard for uploading gallery media and managing enquiries. The production build is generated by Nitro for Cloudflare Workers.

## Errors reproduced from the original ZIP

### 1. `npm ci` failed with `EBADPLATFORM`

Original command:

```bash
npm ci
```

Original error:

```text
npm error code EBADPLATFORM
Unsupported platform for @esbuild/aix-ppc64@0.28.1
wanted {"os":"aix","cpu":"ppc64"}
current {"os":"linux","cpu":"x64"}
```

Cause: the original `package-lock.json` incorrectly marked esbuild platform binaries as normal/extraneous packages instead of optional platform packages.

Fix: corrected the lockfile metadata while keeping the locked dependency versions. A clean `npm ci` then completed successfully.

### 2. TypeScript errors

Original problems:

```text
src/routes/about.tsx: Cannot find name 'friendship'.
src/routes/index.tsx: Cannot find module '@/assets/friendship-tree.asset.json'.
```

Cause: the About page used a variable that was never imported, and the Home page referenced an asset file outside `src/assets`.

Fix: replaced the broken metadata references with actual local image imports already present in `src/assets`.

### 3. Broken image paths outside Lovable

The original code used Lovable-only paths such as:

```text
/__l5e/assets-v1/...
```

It also referenced missing public files such as:

```text
/images/friendship.jpeg
/images/award.jpeg
```

Those paths are not included as real image files in the ZIP, so they would return HTTP 404 after moving the project outside Lovable.

Fix: changed the logo and static gallery/about/home images to use the bundled PNG and JPG files under `src/assets`. Added a local favicon.

### 4. About page server-rendering failure

The missing `friendship` variable could produce:

```text
ReferenceError: friendship is not defined
```

Fix: imported a local image and used that import directly.

### 5. Conflicting Supabase migrations

The July 13 migration set recreated objects already created by the July 4 migration set, including:

- `public.app_role`
- `public.user_roles`
- `public.has_role`
- `public.handle_new_user`
- `on_auth_user_created`
- gallery storage policies

A fresh migration run could stop with errors such as `already exists`.

The current source code and generated Supabase types use the July 4 schema names `gallery_media` and `enquiries`, so the conflicting July 13 migration set was removed.

Important: the remaining migrations create database objects and storage policies, but the `gallery` Storage bucket itself must still be created in the Supabase dashboard.

### 6. `.env` was not ignored

The original `.gitignore` did not exclude `.env`, so local configuration could be committed accidentally.

Fix:

- added `.env` and `.env.*` to `.gitignore`
- kept `.env.example`
- removed the real `.env` from the GitHub-ready copy

### 7. ESLint errors

The original lint run reported hundreds of Prettier formatting errors and two explicit `any` errors in catch blocks.

Fix: formatted the source and changed the catch variables from `any` to `unknown`. The final lint run has zero errors and six non-blocking Fast Refresh warnings in shared UI component files.

### 8. Preview command mismatch

The original package used `vite preview`, while the build is a Nitro/Cloudflare output under `.output`.

Fix: changed the preview script to `nitro preview`, matching the generated target. Nitro preview starts Wrangler, so it may require internet access for Cloudflare-related checks even though the local worker itself can run.

## Verification completed

- `npm ci`: passed

- `npm run build`: passed
- Public routes `/`, `/about`, `/activities`, `/facilities`, `/gallery`, `/contact`, `/auth` and `/admin` returned HTTP 200 in local preview simulation
- `.env` is ignored by Git
- Local Git initialization, staging and commit simulation passed

## GitHub steps

```bash
git init
git add .
git commit -m "Initial Mango Kids project"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY.git
git push -u origin main
```

Do not push the original ZIP as a single file. Extract it and push the project contents. Do not commit `.env`, `node_modules`, `.output` or `.wrangler`.
